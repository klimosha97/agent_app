# Памятка: Как работают перцентили в системе

## Что такое перцентиль простыми словами

Перцентиль — это **место игрока среди всех остальных** на его позиции.

Если у нападающего перцентиль по голам = **0.85** (85%), это значит:
> «Он забивает больше, чем 85% всех нападающих в турнире»

- **0.99** = лучший среди всех (топ-1%)
- **0.80** = лучше 80% игроков — очень хороший результат
- **0.50** = средний уровень
- **0.20** = хуже 80% игроков — слабый показатель

---

## Какие бывают сравнения (4 вида)

### 1. За сезон (SEASON) — стабильность на дистанции

**Что сравниваем:** среднюю статистику игрока за 90 минут за весь сезон
**С кем сравниваем:** со всеми игроками на той же позиции за весь сезон

> Пример: Иванов играет центрального нападающего (Ф Ц). За 15 туров МФЛ его средние PER90 — 0.8 голов.
> Система смотрит на всех нападающих (Ф Ц + Ф ЛЦ + Ф ПЦ) с >200 минутами.
> Допустим, из 25 нападающих 22 забивают меньше → перцентиль = 22/25 = **0.88**

**Зачем:** Показывает, кто стабильно сильнее на своей позиции на протяжении всего турнира. Одна яркая игра не поможет — нужно показывать результат раз за разом.

---

### 2. За тур vs лига (LEAGUE) — яркость в одном матче

**Что сравниваем:** статистику игрока за конкретный тур (один матч)
**С кем сравниваем:** со средними показателями PER90 всех игроков той же позиции за весь сезон

> Пример: В 15-м туре Петров (опорный, ЗП Ц) сделал 8 отборов за матч.
> Среднее PER90 по всем опорникам (ЗП Ц + ЗП Л + ЗП П + ЗП ЛЦ + ЗП ПЦ) за сезон — у 90% из них меньше 8.
> Перцентиль за тур = **0.90**

**Зачем:** Находит игроков, которые выдали выдающийся матч — «вспышку». Даже если в целом за сезон игрок средний, один сильный тур сразу виден.

---

### 3. За тур vs корзина (TIER) — с учётом уровня команды

**Что сравниваем:** статистику игрока за тур
**С кем сравниваем:** только с игроками той же позиции из команд **такого же уровня**

Команды делятся на две корзины:
- **ТОП** — верхняя половина таблицы
- **НИЗ** — нижняя половина таблицы

> Пример: Сидоров играет за команду из нижней половины. Его 4 паса вперёд за тур — это много для команды низа.
> Среди игроков из верхней половины 4 паса — это средний результат.
> Перцентиль Сидорова внутри своей корзины = **0.82**, а по всей лиге был бы = 0.55.

**Зачем:** Даёт шанс талантам из слабых команд. Игрок слабой команды может показывать выдающиеся цифры для своего уровня, но в общем зачёте проигрывать из-за окружения.

---

### 4. Эталонный сезон (BENCHMARK) — сравнение с прошлым годом

**Что сравниваем:** статистику игрока за тур
**С кем сравниваем:** со средними PER90 из загруженного «эталонного» прошлого сезона того же турнира

> Пример: Загружен МФЛ сезона 2024/2025 как эталон. Молодой игрок в 2025/2026 показывает результаты на уровне 90-го перцентиля прошлого сезона — значит, он на уровне лучших прошлого года.

**Зачем:** Понять, тянет ли игрок уровень предыдущих сезонов. Особенно полезно для юношеских лиг, где каждый год состав обновляется.

---

## Правило 200 минут

### В чём проблема без этого правила

Представьте: игрок вышел на замену на 10 минут, сделал 1 гол и 1 ассист. Его PER90 = 9.0 голов и 9.0 ассистов за 90 минут. Это космические цифры, но на самом деле — случайность.

### Как работает правило

| Минут за сезон | Что происходит |
|---|---|
| **> 200 минут** | Игрок полностью в системе: участвует в рейтингах, включён в baseline для сравнения |
| **≤ 200 минут** | Перцентили считаются, но игрок помечен «**без оценки**» и **не попадает в топ-листы** |

### Два эффекта правила

**Эффект 1 — Чистый baseline:**
Когда считаем перцентиль, в знаменатель (всего игроков) и числитель (сколько хуже) попадают только игроки с >200 минутами. Это даёт честное сравнение.

**Эффект 2 — Фильтрация топов:**
Игрок с ≤200 минутами не появится в «Топ за тур» или «Топ за сезон», даже если его цифры космические.

### Пример

| Игрок | Минуты | Голы PER90 | В baseline? | В топе? |
|---|---|---|---|---|
| Иванов | 1200 | 0.7 | Да | Да |
| Петров | 800 | 0.5 | Да | Да |
| Сидоров | 150 | 2.1 (раздуто!) | Нет | Нет — «без оценки» |
| Козлов | 50 | 4.5 (нерелевантно) | Нет | Нет — «без оценки» |

### Начало турнира

Если загружены только 1-2 тура и ни у кого нет >200 минут:
- Baseline пустой → все перцентили = пусто
- Все игроки помечены «без оценки»
- Топ-листы показывают: «Нет данных»
- **Это нормально** — система начнёт работать после 3-4 тура, когда основные игроки наберут время

## Группы сравнения позиций

Вместо сравнения «З ЛЦ только с З ЛЦ» (может быть всего 5 игроков — мало для статистики), используются **7 укрупнённых групп**:

| Группа | Позиции, которые входят | Описание |
|---|---|---|
| **НАП** | Ф Ц, Ф ЛЦ, Ф ПЦ | Все центральные нападающие |
| **АП Ц** | АП Ц, АП ЛЦ, АП ПЦ | Центральные атакующие полузащитники |
| **ФЛ** | АП Л, АП П, П Л, П П | Все фланговые игроки |
| **ПЗ Ц** | П Ц, П ЛЦ, П ПЦ | Центральные полузащитники |
| **ОП** | ЗП Ц, ЗП Л, ЗП П, ЗП ЛЦ, ЗП ПЦ | Все опорные полузащитники |
| **ЦЗ** | З Ц, З ЛЦ, З ПЦ | Все центральные защитники |
| **КЗ** | З Л, З П | Крайние защитники |

**Важно:** метрики (какие параметры оценивать) берутся из **точной** позиции игрока. Но **сравнение** идёт со всей группой. То есть:
- Для игрока «З ЛЦ» берутся метрики, прописанные именно для «З ЛЦ» в POSITION_INFO.txt
- Но его перцентиль считается среди **всех** центральных защитников (З Ц + З ЛЦ + З ПЦ) = группа «ЦЗ»

---

## Три типа метрик: Core, Support, Risk

Для каждой позиции в файле POSITION_INFO.txt прописаны метрики трёх типов:

### Core — главные параметры (вес 70%)
Это то, что **определяет** позицию. Для нападающего — голы, удары, единоборства вверху. Для защитника — отборы, перехваты, воздушные дуэли.

### Support — дополнительные параметры (вес 30%)
Углубляют картину. Для нападающего — xG, ключевые пасы, дриблинг. Для защитника — длинные пасы, продвижение мяча.

### Risk — дисциплина (без штрафа, только пометки)
Жёлтые/красные карточки, фолы, потери мяча, грубые ошибки. **Не снижают** оценку, но отображаются как предупреждения.

---

## Как считается итоговая оценка

### Шаг 1: Берём лучшие core-метрики

Не все core-параметры могут быть одинаково сильными. Система берёт **лучшие 60%** от имеющихся (но минимум 3):

> Пример: у нападающего 8 core-метрик с данными.
> k = max(3, ceil(0.6 × 8)) = max(3, 5) = 5
> Берём 5 лучших перцентилей: 0.95, 0.92, 0.88, 0.85, 0.80
> **core_score** = среднее = (0.95+0.92+0.88+0.85+0.80) / 5 = **0.88**

### Шаг 2: Аналогично для support

> support_score = среднее топ-60% support-перцентилей

### Шаг 3: Поправка на полноту данных (coverage)

Если данных мало, оценка слегка снижается:

> **core_score_adj** = core_score × (0.5 + 0.5 × coverage)
>
> Примеры:
> - Coverage 100% (все метрики есть) → множитель = 1.0 (без штрафа)
> - Coverage 80% → множитель = 0.9
> - Coverage 50% → множитель = 0.75

### Шаг 4: Итоговый балл

> **total_score** = 0.7 × core_score_adj + 0.3 × support_score_adj

Core важнее (70%), support дополняет (30%).

### Шаг 5: Good% — стабильность

> **good_share_core** = доля core-метрик с перцентилем ≥ 0.80

> Пример: из 8 core-метрик 5 имеют перцентиль ≥ 80%
> Good% = 5/8 = **62.5%**

### Чем отличается Core от Good%

| | Core Score | Good% |
|---|---|---|
| **Что считает** | Среднее лучших 60% метрик | Доля всех метрик выше 80-го перцентиля |
| **Что показывает** | Насколько сильны лучшие стороны игрока | Насколько игрок стабильно хорош во всём |
| **Может быть** | Core=99, Good=50% | Суперсильный в паре метрик, но в остальных средний |
| **Или** | Core=75, Good=90% | Не суперзвезда ни в чём, но стабильно хорош везде |

## Флаги качества данных

### «Недостаточно данных» (insufficient_data)
Появляется когда:
- Менее 4 core-метрик с данными, ИЛИ
- Coverage core < 60%

Такой игрок **не попадает** в топ-листы.

### «Без оценки» (insufficient_minutes)
Появляется когда:
- Игрок сыграл ≤ 200 минут за сезон

Такой игрок **не попадает** в топ-листы, и в его профиле показывается красный баннер.

---

## Воронки фильтрации (Funnel)

В топ-листах можно включить фильтр по минимальному total_score:

| Фильтр | Условие | Что видим |
|---|---|---|
| **Все** | без фильтра | Все игроки с достаточными данными |
| **75p+** | total_score ≥ 0.75 | Только хорошие и выше |
| **85p+** | total_score ≥ 0.85 | Только очень сильные |
| **90p+** | total_score ≥ 0.90 | Только элитные |

## Полная схема: что происходит при загрузке данных

```
1. Загружается файл .xlsx (сезон PER90 или тур)
         ↓
2. Данные сохраняются в БД как есть (без изменений!)
         ↓
3. Автоматически запускается расчёт перцентилей:
         ↓
   ┌─────────────────────────────────────────────┐
   │  Для каждого игрока и каждой его метрики:   │
   │                                              │
   │  a) Определяем позицию → группу сравнения   │
   │  b) Берём все игроки той же группы с >200 мин│
   │  c) Считаем: сколько из них хуже? → перцентиль│
   └─────────────────────────────────────────────┘
         ↓
4. Агрегация в оценки:
   - Core score (лучшие 60% core-метрик)
   - Support score (лучшие 60% support-метрик)
   - Total = 0.7 × core + 0.3 × support
   - Good% = доля core ≥ 80%
   - Флаги: insufficient_data, insufficient_minutes, risk
         ↓
5. Результаты доступны в интерфейсе:
   - Топ за тур / за сезон
   - Топ по позициям
   - Профиль игрока с перцентилями
   - Карточка сравнения
```

## Краткая шпаргалка

| Вопрос | Ответ |
|---|---|
| Кто с кем сравнивается? | Игрок сравнивается с игроками той же **группы позиций** (7 групп из 23 позиций) |
| Минимум минут? | **> 200** минут за сезон, иначе «без оценки» |
| Что важнее, core или support? | Core = 70%, Support = 30% |
| Risk снижает оценку? | Нет, только пометка-предупреждение |
| Что такое Good%? | Доля core-параметров, где игрок в топ-20% |
| Почему топы пустые в начале сезона? | Никто ещё не набрал 200 минут — это нормально |
| Можно видеть данные игрока с <200 мин? | Да, в его профиле всё есть, но с пометкой и без рейтинга |
